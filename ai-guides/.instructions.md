Reglas obligatorias (resumen):

- **Arquitectura:** Generar cambios respetando la separación `core` / `adapter` / `api`.
- **Paquetes:** Usar el prefijo `cl.cesarg.siiproxyha` y subpaquetes por capa (`core`, `api`, `adapter.persistence`, `adapter.storage`, `config`).
- **Persistencia:** Acceder a Postgres mediante Spring Data JDBC/JPA según decisión del equipo; preferir JPA si se necesita mapeo complejo.
- **Storage:** Usar adaptador para S3 compatible con MinIO; nunca codificar credenciales en código.
- **Configuración:** Valores sensibles vía variables de entorno y `application.yaml` con placeholders.
- **Tests:** Todo nuevo endpoint debe venir acompañado de tests unitarios y al menos un test de integración que use el `docker-compose` de desarrollo.
- **OpenAPI:** Actualizar `ai-guides/api/openapi.yaml` con cada cambio en endpoints.

Al generar código, la IA debe comentar la decisión arquitectónica en el PR/commit.
